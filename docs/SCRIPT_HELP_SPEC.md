# 脚本帮助信息模板规范

## 大纲顺序与内容

```bash
F_HELP()
{
    echo "
    用途：[脚本功能描述]
    特征码：
        ${GAN_WHAT_FUCK:-'未命名'}
    权限要求：
        ${NEED_PRIVILEGES:-'未指定'}
    依赖：
        ./function.sh
        ./my_conf/env.sh--<NAME>      #--- [依赖说明]
    注意：[注意事项]
    用法：
        $0  -h|--help
        $0  {-n|--name <参数>}  [{-o|--option <参数>}]  [-flag]
    参数规范：
        无包围符号 ：-a                : 必选【选项】
                   ：val               : 必选【参数值】
                   ：val1 val2 -a -b   : 必选【选项或参数值】，且不分先后顺序
        []         ：[-a]              : 可选【选项】
                   ：[val]             : 可选【参数值】
        <>         ：<val>             : 需替换的具体值（用户必须提供）
        %%         ：%val%             : 通配符（包含匹配，如%error%匹配error_code）
        |          ：val1|val2|<valn>  : 多选一
        {}         ：{-a <val>}        : 必须成组出现【选项+参数值】，且保持顺序
                   ：{val1 val2}       : 必须成组的【参数值组合】，且必须按顺序提供
    参数说明：
        -h|--help                    此帮助
        -n|--name <参数>             [参数说明]
        -o|--option <参数>           [参数说明]
    示例:
        $0  -n example
    "
}
```

## 关键要点

- **特征码/权限**：必须使用变量引用的形式。
- **参数规范**：必须原样保留上述规范文本块。
- **用法符号**：
  - `{}`   代表 **必选** 的 **成组** 参数（如 `{-n <val>}`）。
  - `[{}]` 代表 **可选** 的 **成组** 参数（如 `[{-n <val>}]`）。
