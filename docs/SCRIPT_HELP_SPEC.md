# 脚本帮助信息编写规范

## 大纲顺序与内容

```bash
F_HELP()
{
    echo "
    用途：<脚本功能描述>
    特征码：
        ${GAN_WHAT_FUCK:-'未命名'}
    权限要求：
        ${NEED_PRIVILEGES:-'未指定'}
    依赖：
        ./function.sh
        ./my_conf/env.sh--<NAME>      #-- <依赖说明>
    注意：
        <注意事项>
    用法：
        \$0  --help|-h                                        # 帮助
        \$0  {--name|-n <参数>}  [{--option|-o <参数>}]  \\
             [--flag]                                         #-- <简短用途说明>
    参数规范：
        |          : val1|val2|<valn>  : 多选一
        <>         : <val>             : 需替换的具体值（用户必须提供）
        %%         : %val%             : 双边通配符（包含匹配，如'%error%'匹配'os_error_code'）
        %          : val%              : 单边通配符（包含匹配，如'error%'匹配'error_code'，'%error'匹配'os_error'）
        []         : [-a]              : 可选
                   : [-a val val2]     : 可选，必须成组出现，且保持顺序
        {}         : {-a}              : 必选
                   : {-a val val2}     : 必选，必须成组出现，且保持顺序
        无{}[]包围 : -a                : 必选
                   : -a val val2       : 必选，且不分先后顺序
        [{}]       : [{-a val1} val2]  : 总体可选，使用此项时，必选'-a val1'，可选'val2'，必须成组出现，且保持顺序
        {[]}       : {[-a val1] val2}  : 总体必选，使用此项时，可选'-a val1'，必选'val2'，必须成组出现，且保持顺序
        arg1[,...argn] : val1,val2,var3=val3 : 有一个参数或多个参数，用','分隔
    参数说明：
        --help|-h                    # 帮助
        --name|-n <参数>             # <选项参数说明>
        --option|-o <参数>           # <选项参数说明>
        --flag                       # <选项说明>
    示例:
        \$0  -n example      #-- <实现某个功能>
    "
}
```

## 关键要点

- **特征码/权限**：必须使用变量引用的形式。
- **选项参数规范**：必须原样保留上述规范文本块。
- **用法选项参数太多太长**：请换行，以便于用户阅读
